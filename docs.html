What is FarmSimulation?

Real farmers balance scarce capital against volatile markets, fight pests and drought, and time harvests for peak prices — miss one irrigation and a week of growth dies overnight.

FarmSimulation is an OpenEnv-compatible Reinforcement Learning environment that makes agents solve the Agricultural Resource Management problem. It replicates the full crop lifecycle, market economics, and environmental hazards that any autonomous farming agent must navigate to turn a profit.

Quick Start

Get your local dev server running in under 2 minutes.

1. Install & Run Locally

# Install dependencies (uv-compatible)
pip install -e .

# Start the environment server + Gradio UI
uvicorn server.app:app --host 0.0.0.0 --port 7860

# Open the interactive dashboard
open http://localhost:7860

2. Run the LLM Agent

export HF_TOKEN="hf_your_token_here"
export MODEL_NAME="Qwen/Qwen2.5-72B-Instruct"
export FARMING_ENV_URL="http://localhost:7860"
export MAX_STEPS=60

python inference.py
# → saves baseline_results.json

3. Select Task Difficulty

export FARMING_TASK_ID=3   # 1=easy  2=medium  3=hard
python inference.py
// deployment

Docker

# Build the image
docker build -t farming-sim .

# Run with your HF token
docker run -p 7860:7860 -e HF_TOKEN=hf_xxx farming-sim

The Dockerfile uses python:3.11-slim and exposes port 7860. The image is also compatible with Hugging Face Spaces (set HF_TOKEN via Space secrets).

// simulation

Physics Engine

Every day progression runs five simultaneous simulation passes — grounded in agricultural science, not random walks.

1. Precipitation   → aquifer += climate.precipitation × 2   [mm → litres]
2. Moisture Decay  → soil_moisture -= climate.moisture_decay + weed_penalty(0.05)
3. Pest Escalation → pest_severity = min(1.0, (sev + 0.1) × 1.5)   [exponential]
4. Health Damage   → health -= 0.1   [if moisture < 0.2 OR any NPK < 0.2 OR moisture > 0.9]
5. Market Tick    → sell_price = base × (1.0 + 0.2·sin(2π·day/20 + offset) + noise)

Climate System

Climates rotate every 10 days: temperate → arid → tropical. Extreme temperatures (>32°C or <10°C) freeze crop growth entirely.

// economics

Market Dynamics

Each crop rides its own 20-day sine wave, desynchronized by a fixed offset so peaks never align:

offset = {"wheat": 0, "rice": 7, "corn": 13}
sell_price = base_sell × (1.0 + 0.20 × sin(2π × (day + offset) / 20) + noise)

Price Elasticity

Large sell orders crash their own price:

price_drop = min(50%, qty_sold / 10kg × 1%)
sell_price *= (1.0 - price_drop)

// scoring

Grading Formulas

Every episode produces a final grade in [0.0, 1.0] via observation.metadata["grade"]. A score ≥ 0.80 is professional tier.

Task 1 — Single Crop Stable

score = clamp(net_worth / (initial_money × 2.0), 0, 1) − min(0.20, withered × 0.05)

Task 2 — Multi-Crop Market Timing

score = 0.6 × profit_score + 0.4 × timing_score − min(0.30, withered × 0.10)

timing_score = clamp(premium_revenue / (total_revenue × 0.3), 0, 1)

Task 3 — Drought Survival

score = 0.5 × profit_score + 0.3 × survival_score + 0.2 × resilience_score
      − min(0.40, withered × 0.15)

# resilience_score = days where ≥ 2 plots had health ≥ 0.6
resilience_score = healthy_days / max_days

LLM Agent Architecture

inference.py implements a stateless LLM-as-Agent loop compatible with any OpenAI-format API.

System Prompt  ──►  farming strategy rules (one-shot)
                         │
Observation    ──►  text_summary + valid_actions list
                         │
LLM Response   ──►  raw JSON string
                  '{"action_type": "harvest", "plot_id": 2}'
                         │
Parser         ──►  parse_action() → validate_action() → FALLBACK on malformed
                         │
Environment    ──►  env.step(action) → new obs + reward
                         │
History Buffer ──►  last 4 steps kept in context window

API Reference

The server runs on port 7860. All endpoints return JSON.

GET
/health
Liveness check — returns {"status": "ok"}
POST
/reset
Body: {"task_id": 1|2|3} — Resets episode → returns initial FarmObservation
POST
/step
Body: {"action": FarmAction} — Takes one action → returns new FarmObservation
GET
/state
Returns full internal FarmState for debugging
GET
/
Gradio interactive dashboard UI
ℹ️
OpenEnv Protocol: All endpoints are compliant with the OpenEnv Core REST specification. The /reset and /step endpoints follow the standard request/response contract for agent loop integration.
